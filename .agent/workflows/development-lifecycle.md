## 期待されるフロー

このプロジェクトで作業を開始するエージェントは、自己流の操作を行う前に、必ず以下のステップを順に実行（または確認）してください。

### 1. グローバルスキルの初期確認
- `~/.agent/skills/` ディレクトリ配下にある、この作業に関連するスキル（`git-lifecycle`, `pr-workflow` など）を必ず `view_file` で読み込んでください。
- **[Skill: ...]** という形式で、使用するスキルを宣言してください。

### 2. ブランチの準備 (git-lifecycle スキルの適用)
- `git-lifecycle` スキルの指示に従い、`main` ブランチからの分岐と、`main` の最新状態の同期を確認してください。
- 原則として `main` での直接作業は禁止です。

### 3. 実装と検証
- コード変更後は、必ず以下のコマンドで品質を確認してください。
  ```bash
  dart test
  ```

### 4. プルリクエストの作成 (pr-workflow スキルの適用)
// turbo
- `pr-workflow` スキルの指示を厳守してください。
- **重要**: `gh pr create --body "..."` を直接実行せず、必ず `.github/PULL_REQUEST_TEMPLATE_TEMP.md` 等の一時ファイルを作成し、`--body-file` オプションを使用してください。
- PR作成後は、作成に使用した一時ファイルを削除してください。
